Mobile — Transaction Modal
Design Notes
The modal opens over the Money Snapshot screen with a backdrop blur. It pre-fills the category name and emoji, defaults to the correct direction (Expense for rent, Income for salary), and pre-selects the account created in Section A (shown with a green "Auto-selected" badge). The user just needs to confirm/enter the amount and pick the day of month. On mobile, the modal slides up from the bottom as a native-feeling bottom sheet with a drag handle. The "Add Rent 🏠" CTA uses the category emoji for personality.

Mobile — Family Life
Design Notes
The activity card now includes a day-of-week multi-select row — seven chips (Mon–Sun), tap to select multiple days. This captures activities that happen on multiple days (e.g. Piano on Wed + Sat). On mobile the chips use single-letter abbreviations (M, T, W, T, F, S, S) to fit the narrower width. The rest of the card (time, frequency, who, pickup) remains in the compact 4-column grid. Discover teasers use a 3×2 grid on mobile instead of 5-across — more tap-friendly and less cramped. The closing message strip is retained as the emotional bridge to the completion screen.

Changes in v2
1. Bank dropdown replaces owner avatars on the account row — more practical, less cluttered. Owner can be set later or defaults to the logged-in user.

2. "Add a regular transaction" replaces "Your regulars" — clearer intent. Tapping a category opens a focused modal rather than trying to inline-edit within the crowded main screen.

3. Transaction modal pre-fills category, defaults direction (expense/income), pre-selects the account from Section A. User just confirms name, amount, and day. Mobile version uses a bottom sheet pattern.

4. Day-of-week multi-select chips added to the activity card — Mon through Sun, tap multiple. Supports activities on multiple days. Mobile uses single-letter abbreviations.

5. Mobile frames shown alongside desktop for every step — demonstrating how the layout adapts. Mobile uses stacked fields, abbreviated labels, compact chips, and bottom-sheet modals.
