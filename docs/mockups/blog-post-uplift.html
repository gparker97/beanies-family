spec & open calls

what's lifting from where, what's preserved, what's NOT done.

What's new in v1

  • Per-issue tint inherited from /blog index. Compute issue number the same way the index does (chronological ascending position) and pass the tint into [...slug].astro as a CSS variable scope. 5-tint cycle: orange → terracotta → sky → kraft → slate.
  • Hero wash strip. 320px tint gradient behind the kicker + title + subtitle, fading to cloud-white before the body prose starts. Includes a faint paper-grain noise overlay (same recipe as the masthead's masthead-wash::after).
  • Kicker pill. "issue 04 · 17 apr · 4 min read" — Outfit uppercase tracked, with a tint dot. Mirrors the masthead's masthead-issue-stamp vocabulary.
  • Bigger title. Up from 1.75rem → clamp(1.6, 3.8vw, 2.4rem). Tighter letter-spacing.
  • Fresh-divider echo. Tint-colored orange-dot ornament between hero and prose. Same family as the index's "fresh off the press" divider, sized down.
  • Tint-aware prose details. Links, h2 prefix-dot, blockquote rule + wash, image frame border, prev/next hover border — all use the post's tint.
  • Single drop cap. Fraunces italic, on the very first paragraph only. One editorial flourish; doesn't repeat per section.
  • Fraunces blockquote. The only piece of serif body copy on the page. Reads as quoted from the page.

What's preserved

  • Sans-serif body prose. Outfit at 15.5px, line-height 1.78. Journal voice, not magazine voice. Serif body would push this into the guides' territory.
  • Fraunces italic subtitle. Already doing real work; no change needed.
  • Lightbox, breadcrumbs, prev/next nav, further-reading, Substack subscribe — all kept as-is. Visual styling adapts to the tint; structure unchanged.

What's NOT in v1 (and why)

  • Reading progress bar. That's the guide page's signature. Lifting it would flatten the blog/guide distinction.
  • Numbered or decorated h2s. Guide vocabulary. Blog h2 gets only a small tint dot.
  • Drop caps on every section. Too decorative; one drop cap on the first paragraph is the whole budget.
  • Mascots / illustrations on the detail page. The index handles mascot heavy-lifting. Detail pages stay text-first.
  • Per-tint cover plate. If a post has no coverImage, the body just starts. We don't render a blank tint plate. (The index handles cover-less posts with the wobble emoji; the detail doesn't need to.)
