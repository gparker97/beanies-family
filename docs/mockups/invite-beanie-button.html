Invite Beanie button — three directions

The button currently sits as a low-contrast secondary action next to "Add Bean". Inviting is the central gesture that makes the platform work, so the button should feel like an invitation, not an admin task. Three variants below — each shows the button in the actual pod-page header context next to "Add Bean" for visual weighting.

Design considerations

  • Brand integrity — all three stay strictly within the CIG palette (Heritage Orange, Terracotta, Golden as the warm accent — no new tints). Mascots reused as-is per the never-recolor rule (drop-shadow + opacity only).
  • Reduced motion — every animated variant will gate animations behind @media (prefers-reduced-motion: reduce) when implemented, so users with vestibular sensitivities aren't strobed.
  • Same hierarchy on mobile — on phones the two buttons stack to full-width rows; the visual weight difference still reads (Invite has a halo / sheen / mascot, Add Bean stays a flat gradient).
  • Why not Sky Silk in the gradient? Cool blues fight the warm "invitation" tone. Gold (`#F4B942` from CIG) gives the same depth without breaking the gesture.
  • Pairs with FAQ link fix — anchors in the FAQ (Family Link, accounts.google.com) get the standard text-primary-500 hover:underline treatment. This is a separate bug — the existing .wizard-faq-link class lives in scoped CSS but the FAQ content is rendered via v-html, which scoped styles never reach.
