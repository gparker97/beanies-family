ii.

does the stack get any taller?

Almost not at all. The text card sits beside the bean, not below it, so the bean still drives row height. The cost is horizontal — the stack is wider and may need to flip sides when the active tab is on the screen edge.

three live tests · auto side-flip

final spec & open calls

what changes from v2, and what's still up to greg.

What's the same as v2

  • Pattern law unchanged. Cluster bloom = ADD, pure bean stack = NAVIGATE. The ＋ seal still distinguishes FAB beans from nav beans.
  • Cap of 6 items. Same height budget; bean still drives row height.
  • Active route at the bottom. Closest to the thumb, ringed in Heritage Orange.
  • Scrim, Esc, back-gesture, same-tab tap. All behave identically.

What's new in v3

  • Side card per bean. Outfit 13/600 label + Inter 10.5/400 hint. Pointer notch faces the bean.
  • Bean stays anchored over the tab. The stack positions by bean centre, not row centre.
  • Auto side-flip. The card extends toward whichever side of the screen has more horizontal space. Tab in the left half → card on the right; right half → card on the left. JS sets the side class on open and on resize.
  • Eyebrow ribbon dropped. The active tab already names the category, so the floating "🌳 planning" pill above the stack was redundant. Bonus: ~30px of extra vertical headroom inside the stack column.
  • Hint copy ≤ 22 chars. Soft target; never wraps; ellipsis fallback for translations. See copy guide above.
  • ＋N more bean carries a hint too. Reads "milestones, photos & 2 more" so the user doesn't have to tap to see what's hidden.

Open calls

  • Hint copy review. The 18 strings in the copy guide are first-pass — greg's voice should pass over them before they ship to uiStrings.ts.
  • Hint typeface. Currently Inter 10.5 for functional copy. Alternative: Fraunces italic 11 for an editorial feel. Trade-off is legibility vs character.
  • Pointer notch. Tiny rotated square aimed at the bean. Small visual flourish; could drop entirely if it feels fussy on phone-sized scale.
  • i18n length. Chinese hint translations are typically shorter; German/French may push past the cap. The card has max-width: 168px + ellipsis as a safety net.
