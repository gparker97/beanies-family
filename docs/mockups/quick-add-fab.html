asset · FAB iconography

Gemini concept vs hand-authored SVG

The Gemini PNG is illustrated and warm but its subtle shading collapses into muddy pixels at 60 px (the actual FAB size). A clean geometric SVG — same peek-a-boo composition, simpler shape language — reads sharper at small size, weighs ~2 KB, and themes via a single CSS variable.

  • Matches the Gemini peek-a-boo composition — head pops above the plus, two hands grip the crossbar top.
  • Circle background fills via --fab-bg — swap orange ↔ slate per theme without shipping two files.
  • Character stays warm across themes (orange bean + blue beanie); only the outer frame changes.
  • Plus arms are all equal — top hidden by the character, but the visible "T" shape still reads as a proper plus sign.

my take

Select a variant on the left to see the breakdown here.

the context that's changed

Issue #37 was filed when the app had 5 addable entities (account, tx, recurring, asset, goal). Today the app has 15+: add activities, todos, trips, sayings, favorites, notes, recipes, cook-logs, medications, allergies, emergency contacts.

A 5-slot arc fan no longer captures reality. That changes the shape of the answer more than the animation does.

my recommendation — built as variant 06

Selected by default. Tap the peek-a-boo beanie on the phone to see it: six everyday beans (Activity, To-do, Transaction, Trip, Cook log, Saying) jar-pop into a 3×2 grid at the top, then Family / Money / Care fade in below. FAB swaps to the slate version in dark mode. Icons pulled from the app's canonical conventions — navigation.ts, BeanTabs.vue, VacationWizard.vue.

The top tier is ordered by how often real families hit add, not by domain. "Accountant-brain" wanted Money up top; "parent-brain" wants the one-tap capture for the soccer schedule change or the thing the 5-year-old just said.

The draggable variant (05) is fun — drag the orange puck to feel it — but I'd ship it as a Settings toggle, not a default.

the FAB itself — peek-a-boo beanie

The button swaps a generic gradient-+ for a beanie peeking over a plus sign (your Gemini concepts). Orange by default, slate-blue in dark mode. Idle bob sells the "peeking" — 2.6s gentle rise-and-settle, killed by prefers-reduced-motion. Tap rotates it 45° with a spring so the beanie tilts into a close affordance.

prod follow-up The source PNGs are 5 MB each — fine for the mockup, too heavy for the app. Before shipping we'll want either SVG (~15 KB, stays sharp at any size, can be restyled via CSS vars) or a 2x/3x optimised PNG set (~40 KB). At 60 px the face may lose detail; the SVG pass would be the place to simplify the micro-features.

hard constraints

  • mobile FAB must sit above the MobileBottomNav with bottom: calc(env(safe-area-inset-bottom) + 92px).
  • z-index FAB and menu above page content, below modals, below toasts.
  • a11y menu closes on Esc, focus traps while open, FAB itself is a <button>.
  • motion all animations gated on prefers-reduced-motion.

drag-to-move, honestly

Apple's AssistiveTouch works because it has to overlay the entire OS. In a single app the payoff is smaller — users want it somewhere predictable. If we ship it, treat it as a power-user toggle in Settings, not a default. I built a working draggable variant so you can feel it.
