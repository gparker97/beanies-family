# Plan: Standardize Text Casing Across beanies.family

> Date: 2026-03-01

## Context

The app had inconsistent text casing â€” page headers, button labels, chip labels, nav items, and settings labels used a mix of Title Case, Sentence case, and lowercase. Additionally, page titles were broken: `AppHeader.vue` read `route.meta.title` (always empty) while the router only set `meta.titleKey`. Four pages worked around this by rendering their own `<h1>` titles below the header.

## Approach

Established a site-wide casing standard:

| Category                                              | Rule                              | Example                                                 |
| ----------------------------------------------------- | --------------------------------- | ------------------------------------------------------- |
| Page headers (AppHeader)                              | all lowercase                     | `family planner`, `good evening, greg`                  |
| Section titles, sidebar headers, form labels          | source lowercase, CSS `uppercase` | stored as `the treehouse`, displayed as `THE TREEHOUSE` |
| Non-sentence text (buttons, chips, labels, nav items) | all lowercase                     | `add member`, `weekly`, `save`                          |
| Full sentences (with period/complete thought)         | sentence case                     | `No transactions yet. Add one to get started.`          |

Rule of thumb: if it has no period, it's lowercase.
